# Checklist de Evidencia

## Capturas sugeridas

1. Pantalla inicial con `Seed-san` cargado y el panel tecnico visible.
2. Cambio a `VRM1 Constraint Twist Sample` mostrando el selector de modelo.
3. Cambio a `Alicia Solid` mostrando creditos y licencia.
4. Boton `Saludar` activo.
5. Boton `Asentir` activo.
6. Boton `Senalar` activo.
7. Expresion `Alegria` activa.
8. Expresion `Tristeza` activa.
9. Expresion `Duda` activa.
10. Reproduccion de audio con `Hablar (audio)`.
11. Panel tecnico donde se vean triangulos, vertices y presets resueltos.

## Guion corto para video

1. Abrir la aplicacion.
2. Mostrar `Seed-san` y explicar que es el agente de orientacion.
3. Activar `Saludar` y `Alegria`.
4. Reproducir audio para mostrar lip-sync.
5. Cambiar a `VRM1 Constraint Twist Sample`.
6. Activar `Asentir` y `Senalar`.
7. Cambiar a `Alicia Solid`.
8. Activar `Duda` y luego reproducir audio.
9. Mostrar el panel tecnico y las fuentes/licencias.

## Checklist final antes de entregar

- `npm run build` ejecutado sin errores.
- `npm run verify:assets` ejecutado sin errores.
- Los 3 modelos cargan en la app.
- Las 3 expresiones se activan por interfaz.
- Las acciones visibles responden correctamente.
- El audio mueve la boca del modelo.
- Se citan fuente y licencia del modelo usado.
- El informe se exporta a PDF.
- El PDF corresponde al archivo [docs/informe-entrega.md](docs/informe-entrega.md).
- El README incluye el enlace no listado de YouTube.
- El video muestra los tres modelos funcionando dentro del motor grafico.
